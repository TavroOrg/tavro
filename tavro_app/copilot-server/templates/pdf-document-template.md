# Tavro PDF Document Template

This is the master reference for all PDFs generated by the Tavro platform.
Every AI-generated document — whether from the AI Assistant, Agent Playground, or Agent
Artifact generation — must follow the format defined in this file.

## Visual Format (applies to all PDFs)

All Tavro PDFs share the same visual style as the Tavro Agent BizOps dashboard:

**Header (dashboard light theme, full-width):**
- Background: white `#ffffff` (rgb 255 255 255), height 44mm
- Bottom border: slate-200 `#e2e8f0` (rgb 226 232 240), 0.3mm
- No accent or divider line inside the header.
- Tavro logo left-aligned inside a white logo tile with subtle slate border
- **Name**: agent name or user request, slate-800 `#1e293b`, bold, 18pt, Y=18
- **Type**: document type label, slate-600 `#475569`, normal, 11pt, Y=25
- Subtitle: "Tavro AI Governance Platform", slate-500, 8pt, Y=31
- Generation date: slate-500, 7.5pt, right-aligned, Y=31

**Document type values by source:**
| Source | Name | Type |
|--------|------|------|
| AI Risk Assessment | Agent name | AI Risk Assessment Report |
| Agent Artifact — Requirements | Agent name | Requirement Document |
| Agent Artifact — Technical Design | Agent name | Technical Document |
| Playground | Agent name | Based on context of user question |
| AI Assistant (single response) | User's question (first 100 chars) | Based on context of user question |
| AI Assistant (conversation export) | Session title | AI Conversation |

**Footer (light gray, full-width, every page):**
- Background: `#f8fafc` (rgb 248 250 252), height 12mm from bottom
- Top border line: `#e2e8f0` (rgb 226 232 240)
- Left: "Tavro AI Governance Platform  ·  Confidential", slate-400, 7pt
- Right: "Page X of Y", slate-400, 7pt

**Content area:**
- Starts at 48mm from top on page 1 (below the 44mm header + 4mm gap)
- Starts at 22mm from top on pages 2+
- Stops 16mm from bottom (above the 12mm footer)
- Left/right margins: 20mm each, content width 170mm

## Global Formatting Rules

- Start directly with one `#` title heading. No preamble before it.
- Use `##` for main sections and `###` for sub-sections.
- Use `**bold**` only for key terms, named entities, and critical values.
- Use `-` for unordered lists and `1.` for numbered steps.
- Use markdown pipe tables for all structured or comparative data.
- Use ASCII characters only — no emojis, curly quotes, en/em dashes, or decorative symbols.
- Do not open with "Here is", "Sure", "Below is", or any acknowledgement.
- Do not close with "I hope this helps", "Let me know", or any sign-off.
- Do not leave `{{...}}` placeholders in the final output.

---

## Global Content Rules

- Derive, do not invent. Every factual claim must come from available context: agent
  metadata, Company Blueprint, risk assessment data, or explicit user input.
- If information is genuinely missing, write "Not specified" — do not omit the section.
- If a Company Blueprint is present, use its dimensions:
  - `[strategy]` for objectives, value, and priorities
  - `[risk]` and `[process]` for constraints and problem statements
  - `[technology]` and `[integration]` for architecture and implementation
  - `[industry]` and `[region]` exactly as provided
- Keep sections concise and specific. Avoid generic filler.
